01. 问题定义与总体结论

推荐定义：这个问题应被定义为「受控多视角渲染外观匹配」，而不是普通的图片相似度或语义相似度。

已知条件

  • 模型 mesh 不变
  • 姿态、朝向可固定
  • 相机距离、尺寸、分辨率可固定
  • 背景可透明
  • 可自动多视角截图

主要目标

  • 评估材质迁移后是否接近 Unity 参考效果
  • 识别颜色、亮度、高光、细节差异来源
  • 为后续自动优化算法提供 objective function

不建议作为主指标

  • CLIP：语义太粗，无法区分材质细节
  • 单纯 MSE/PSNR：和人眼感知不一致
  • 整图 RGB 差：会被透明边缘和空背景污染
ΔE00 色差 MS-SSIM 明暗结构 Highlight 高光反射 LPIPS 感知差异 Gradient 细节纹理 Mask IoU 对齐检查

02. 图像预处理与标准化

你已经实现了透明背景、固定相机与固定像素尺寸，因此不需要复杂的图像配准。但仍建议保留少量标准化步骤，目的不是“修图”，而是保证指标公平、稳定、可复现。

2.1 Alpha mask 有效区域

valid_mask = alpha_ref > threshold AND alpha_candidate > threshold core_mask = alpha_ref > 0.95 AND alpha_candidate > 0.95 edge_mask = 0.05 < alpha < 0.95 full_mask = alpha_ref > 0.05 OR alpha_candidate > 0.05
区域用途原因
core_mask颜色、材质、高光、结构主指标避免抗锯齿边缘污染材质评价
edge_mask轮廓、边缘一致性诊断透明边缘在不同引擎中可能差异较大
full_maskMask IoU、bbox 检查确认相机、尺寸、姿态是否一致

2.2 颜色空间固定

感知颜色指标

推荐路径：

sRGB → CIE Lab → CIEDE2000 ΔE00

适合衡量人眼可感知的颜色差异。

亮度/能量指标

推荐路径：

sRGB → linear RGB → luminance Y

适合高光强度、亮度误差、梯度误差。

2.3 对齐诊断，不一定自动矫正

如果相机严格一致，可以不做图像 warp/resize，但建议每次输出 sanity check：

bbox_center_error

04. 学术研究中的对应范式

该问题对应多个学术方向的交集。它和「从真实图片估计材质」不同，但和 inverse rendering 中的 image reconstruction objective 非常接近。

Inverse Rendering / Material Estimation

常见固定项：geometry、camera pose、illumination、renderer、resolution、color space、tone mapping、material parameterization。

常见 loss：RGB L1/L2、SSIM、VGG perceptual loss、mask/silhouette loss、normal/depth loss、material regularization。

Differentiable Rendering

典型目标函数：

L = image_reconstruction_loss + silhouette_loss + perceptual_loss + regularization_loss

你的 mesh 已固定，因此 silhouette loss 更适合作为对齐检查，而不是材质优化主目标。

Perceptual Image Quality

常见指标：PSNR、SSIM、MS-SSIM、FSIM、GMSD、LPIPS、DISTS、PieAPP、FID/KID。

单图逐视角比较不适合 FID/KID；CLIP 语义太粗，不宜作为主指标。

Material Appearance Matching

特别关注 base color、roughness、metallic、specular、normal map、highlight shape、BRDF lobe width、view-dependent reflectance。

因此必须引入高光区域指标和多视角聚合。

05. 评价指标体系 v2：从工程分数到研究标准

新版结论：本项目不应只使用一个「图片相似度」指标，而应采用「有效性检查 → 科学主指标 → 优化目标 → 诊断指标」的分层体系。总分用于排序，分项指标用于解释，诊断通道用于指导 shader 参数优化。

目前代码中的 RGB MAE、weighted_mae、SSIM 和 human_accept_score 对工程调参有帮助，但还不是足够客观、标准、可复现的研究指标。更合理的做法是借鉴色彩科学、图像质量评价、渲染图误差评价和逆向渲染研究中的成熟思路，再结合本项目的特殊约束：同一 mesh、固定相机、透明背景、多视角、跨 Unity/Laya 引擎、目标是视觉外观匹配而不是物理材质恢复。

5.1 为什么不能只看一个分数

一个总分无法解释错误来源

同样是总分低，原因可能完全不同：整体颜色偏蓝、阴影偏暗、高光过宽、反射方向不对、法线细节太强、透明边缘错位，或者只是视角/截图未对齐。

优化器需要可解释信号

如果只给优化器一个黑盒分数，它不知道该优先调 u_Color、u_AoPower、u_SpecularPower 还是 u_Contrast。分层指标能把视觉差异映射到参数组。

5.3 我们采用的四层指标结构

Validity 有效性检查
先判断截图、透明区域、视角、bbox 是否可靠。失败时不能把误差归因于材质。

使用 ΔE00、luminance、SSIM-L、highlight、gradient 等可复现指标描述外观差异。
Research Score 研究总分
由科学主指标归一化后聚合，作为论文/报告/回归测试的主分。
Diagnostic Channels 诊断通道
按 shader 参数组解释错误来源，指导 base/shadow/specular/reflection/emission 等参数调整。

5.4 有效性检查：先判断这张图能不能被评分

这层不评价材质好坏，只负责判断本次渲染是否可信。它应使用透明 RGBA 的 alpha，而不是历史版本的角点背景检测。

core_mask = alpha_ref > 0.95 AND alpha_candidate > 0.95 full_mask = alpha_ref > 0.05 OR alpha_candidate > 0.05 edge_mask = full_mask - core_mask validity: mask_iou bbox_center_error_px bbox_scale_error foreground_area_ratio alpha_edge_diff view_pose_valid
注意：如果 mask_iou 或 bbox_error 不合格，应标记该视角为 invalid。此时 SSIM、ΔE00、highlight loss 都不能被当成材质差异解释。

5.6 当前实现的完整指标公式清单

本小节对应当前代码中的 material_fit/vision/research_metrics.py。所有图像先被转换为 RGBA float，RGB 和 alpha 均归一化到 [0, 1]。下面公式中的 R 表示 Unity reference，C 表示 Laya candidate，M 表示参与主材质计算的 core_mask 像素集合，|M| 表示集合内像素数量。

5.6.1 Mask 与有效性检查

这组指标不判断材质好坏，只判断“这张图是否能被拿来评价材质”。如果透明遮罩、包围盒或视角明显不一致，后续颜色/高光/细节误差很可能来自截图链路，而不是材质参数。

如果 alpha 有有效信号： full_ref = alpha_ref > 0.05 full_cand = alpha_cand > 0.05 full_mask = full_ref OR full_cand core_mask = (alpha_ref > 0.95) AND (alpha_cand > 0.95) edge_mask = full_mask AND NOT core_mask 如果 alpha 全不透明： 优先使用 explicit_mask；其次使用 fallback_mask；否则使用整图 full_image core_mask = mask > 0.50 full_mask = mask > 0.05
字段公式检测什么当前阈值/说明

5.6.2 Color Accuracy：颜色准确性

这组指标检测固有色、色相、饱和度和主贴图颜色响应是否接近。当前实现先把 sRGB 转为 linear RGB，再转 CIE XYZ 和 CIE Lab，然后计算 CIEDE2000 色差。ΔE00 是色彩科学中常用的人眼感知色差公式，比直接 RGB MAE 更适合判断“颜色看起来差多少”。

sRGB → linear RGB: linear = rgb / 12.92, if rgb <= 0.04045 linear = ((rgb + 0.055) / 1.055) ^ 2.4, otherwise linear RGB → XYZ: [X,Y,Z]^T = M_sRGB_to_XYZ * [R,G,B]^T XYZ → Lab: f(t) = cbrt(t), if t > 216/24389 f(t) = (24389/27 * t + 16) / 116, otherwise L = 116 * f(Y/Yn) - 16 a = 500 * (f(X/Xn) - f(Y/Yn)) b = 200 * (f(Y/Yn) - f(Z/Zn)) CIEDE2000: C1 = sqrt(a1^2 + b1^2), C2 = sqrt(a2^2 + b2^2) C_bar = (C1 + C2) / 2 G = 0.5 * (1 - sqrt(C_bar^7 / (C_bar^7 + 25^7))) a1' = (1 + G) * a1, a2' = (1 + G) * a2 C1' = sqrt(a1'^2 + b1^2), C2' = sqrt(a2'^2 + b2^2) h1' = atan2(b1, a1'), h2' = atan2(b2, a2') ΔL' = L2 - L1 ΔC' = C2' - C1' ΔH' = 2 * sqrt(C1' * C2') * sin(Δh' / 2) L_bar' = (L1 + L2) / 2 C_bar' = (C1' + C2') / 2 h_bar' = circular_mean(h1', h2') T = 1 - 0.17*cos(h_bar' - 30°) + 0.24*cos(2h_bar') + 0.32*cos(3h_bar' + 6°) - 0.20*cos(4h_bar' - 63°) S_L = 1 + 0.015*(L_bar' - 50)^2 / sqrt(20 + (L_bar' - 50)^2) S_C = 1 + 0.045*C_bar' S_H = 1 + 0.015*C_bar'*T R_T = -sin(2*Δθ) * 2 * sqrt(C_bar'^7 / (C_bar'^7 + 25^7)) ΔE00 = sqrt((ΔL'/S_L)^2 + (ΔC'/S_C)^2 + (ΔH'/S_H)^2 + R_T*(ΔC'/S_C)*(ΔH'/S_H)) 当前输出： mean_deltaE00 = mean(ΔE00[p] for p in core_mask) p95_deltaE00 = percentile(ΔE00[p], 95) median_deltaE00 = percentile(ΔE00[p], 50) max_deltaE00 = max(ΔE00[p]) rgb_bias = mean(C_rgb[p] - R_rgb[p]) lab_bias = mean(C_lab[p] - R_lab[p])

mean_deltaE00 反映整体颜色差；p95_deltaE00 用于发现少数区域的严重偏色；rgb_bias 和 lab_bias 是带方向的偏差，能告诉我们 Laya 是整体偏亮、偏红/绿，还是偏黄/蓝。

5.6.3 Luminance & Structure：亮度与结构

这组指标检测整体明暗、阴影、AO、体积感和大块结构是否一致。当前亮度使用 linear RGB 计算，避免直接在 sRGB 上比较亮度导致感知和能量关系混乱。

linear luminance: Y = 0.2126 * R_linear + 0.7152 * G_linear + 0.0722 * B_linear 亮度误差： diff_Y[p] = Y_C[p] - Y_R[p] luminance_mae = mean(abs(diff_Y[p]) for p in core_mask) luminance_bias = mean(diff_Y[p] for p in core_mask) p95_luminance_abs_error = percentile(abs(diff_Y[p]), 95) SSIM-L： full_ssim, ssim_map = structural_similarity(Y_R, Y_C, data_range=1.0, win_size=min_odd(7,W,H)) ssim_l = sum(ssim_map[p] * core_mask[p]) / sum(core_mask[p])

luminance_mae 越大表示整体明暗差越大；luminance_bias 保留方向，正值表示 Laya 更亮，负值表示 Laya 更暗；SSIM-L 在亮度图上计算结构相似度，越接近 1 表示明暗结构越一致。当前代码调用 skimage.metrics.structural_similarity，窗口最大为 7，并在 core_mask 内加权平均 SSIM map。

5.6.4 Highlight / Reflection：高光与反射

高光指标用于检测高光颜色、亮度峰值、面积和反射强弱是否一致。它不是所有材质都启用，因为白色主体、发光纹理、贴图亮部和真正高光容易混淆。当前实现先用参考图亮度判断是否存在可评价高光。

ref_values = Y_R[p] for p in core_mask threshold = percentile(ref_values, 95) contrast = percentile(ref_values, 99) - percentile(ref_values, 80) highlight_mask = core_mask AND (Y_R >= threshold) area_ratio = |highlight_mask| / |core_mask| enable_highlight = contrast > 0.08 AND 0.002 <= area_ratio <= 0.35 AND |highlight_mask| >= 16 如果启用： highlight_deltaE00 = mean(ΔE00[p] for p in highlight_mask) highlight_luminance_mae = mean(abs(Y_C[p] - Y_R[p]) for p in highlight_mask) cand_highlight_mask = core_mask AND (Y_C >= threshold) highlight_area_error = abs(|cand_highlight_mask| - |highlight_mask|) / max(|highlight_mask|, 1) peak_luminance_error = abs(percentile(Y_C[highlight_mask], 99) - percentile(Y_R[highlight_mask], 99))

这些指标主要用于判断 roughness、specular、metallic、reflection、rim 等参数是否造成高光过亮、过暗、过宽、过窄或颜色不对。若启用条件不满足，该视角的高光项标记为 not_applicable，不会强行惩罚没有高光的材质。

5.6.5 Detail / Texture：细节与纹理

这组指标检测纹理清晰度、法线强度、局部对比度和边缘锐度。为了减少透明边缘影响，当前会先对 core_mask 做一次 4 邻域腐蚀，得到更保守的 interior mask；如果腐蚀后为空，则退回使用原始 core_mask。

interior = erode_4_connected(core_mask, iterations=1) if |interior| == 0: interior = core_mask gradient magnitude: grad(Y) = sqrt((dY/dx)^2 + (dY/dy)^2) gradient_loss = mean(abs(grad(Y_R)[p] - grad(Y_C)[p]) for p in interior) Laplacian: lap(Y)[x,y] = Y[x-1,y] + Y[x+1,y] + Y[x,y-1] + Y[x,y+1] - 4 * Y[x,y] laplacian_loss = mean(abs(lap(Y_R)[p] - lap(Y_C)[p]) for p in interior)

gradient_loss 对边缘强度和局部对比更敏感；laplacian_loss 对更高频的纹理锐度、法线细节和振铃更敏感。它们能辅助判断贴图、法线、mipmap、锐化或局部对比设置是否有问题。

5.6.6 Perceptual Optional：感知辅助指标

6.1 单视角归一化误差：当前代码实际公式

为了把不同量纲的指标合成为一个 loss，当前实现先把每个误差归一化到 [0, 1]。这里的分母不是物理常数，而是工程阈值：达到该阈值附近时，说明该项误差已经足够严重，应接近满惩罚。

clamp01(x) = min(max(x, 0), 1) # P0：稳定主指标 E_color_mean = clamp01(mean_deltaE00 / 10.0) E_color_p95 = clamp01(p95_deltaE00 / 20.0) E_luma_mae = clamp01(luminance_mae / 0.20) E_luma_bias = clamp01(abs(luminance_bias) / 0.15) E_struct = clamp01((1.0 - ssim_l) / 0.15) # 仅当 SSIM-L 可用 # P1：高光条件项，只有 highlight.enabled == true 才参与 E_highlight = clamp01( 0.45 * clamp01(highlight_deltaE00 / 20.0) + 0.35 * clamp01(highlight_area_error / 0.50) + 0.20 * clamp01(peak_luminance_error / 0.25) ) # P1：细节纹理项，gradient/laplacian 均可用才参与 E_detail = clamp01( 0.60 * clamp01(gradient_loss / 0.25) + 0.40 * clamp01(laplacian_loss / 0.50) ) # P2：当前只输出，不进入 research_loss_v1 flip_like_error, lpips, dists = report_only

6.2 单视角 research_loss_v1：当前代码实际权重

当前 research_loss_v1 使用 P0 + P1。P2 的 FLIP-like、LPIPS、DISTS 只用于人工查看和报告，不参与优化目标。若某项不可用，例如 SSIM-L 依赖缺失或该视角没有有效高光，则删除该项并对剩余项重新归一化权重。

raw_components: color_mean = E_color_mean color_p95 = E_color_p95 luminance_mae = E_luma_mae luminance_bias = E_luma_bias structure_ssim_l = E_struct if ssim_l is available highlight = E_highlight if highlight.enabled detail_texture = E_detail if gradient_loss and laplacian_loss are available base_weights before renormalization: color_mean = 0.35 color_p95 = 0.15 luminance_mae = 0.20 luminance_bias = 0.10 structure_ssim_l = 0.20 highlight = 0.12 detail_texture = 0.06 total_weight = sum(weight[k] for k in available raw_components) normalized_weight[k] = weight[k] / total_weight view_loss = sum(raw_components[k] * normalized_weight[k]) view_score = 100 * (1 - clamp01(view_loss))
为什么这样设计：P0 指标保证每次运行都可复现；高光只在参考图确实存在可评价高光时参与；细节纹理作为 P1 低权重进入 loss；LPIPS/DISTS 这类模型指标先作为人工复核，避免依赖下载模型、模型版本或运行环境影响主优化闭环。

6.3 多视角聚合：当前代码实际公式

passed_items = views where: status == "ok" AND validity.passed == true AND view_loss is finite all_ok_items = views where: status == "ok" AND view_loss is finite scored_items = passed_items if passed_items is not empty else all_ok_items score_uses_invalid_views = true if passed_items is empty and all_ok_items is not empty mean_loss = mean(view_loss over scored_items) p90_loss = percentile(view_loss over scored_items, 90) max_loss = max(view_loss over scored_items) final_loss = clamp01(0.65 * mean_loss + 0.20 * p90_loss + 0.15 * max_loss) final_score = 100 * (1 - final_loss)
为什么不能只取平均？ 因为某些角度可能高光、反射或法线表现很差，简单平均会掩盖最差视角。加入 p90/max 惩罚可以避免“某一面完全不像但总分还不错”。如果所有视角都没通过有效性检查，系统仍会输出诊断分，但会标记 score_uses_invalid_views，UI 中应显示“不可采信”。

07. 自动优化算法体系：从调参工具到材质外观搜索系统

目标修正：优化算法的目标不是“可解释地跑通”，而是在真实 Laya 渲染闭环中，自动搜索出接近 Unity 参考和用户预期的材质效果。可解释性、分组和规则只是增强自动优化能力的工程手段，不能替代最终效果。

本项目的算法问题本质上是一个跨引擎、强耦合、昂贵评估、无可靠梯度的 black-box appearance optimization。输入是 Unity shader、Unity 材质参数、参考多视角图和 Laya shader/lmat；输出应该是可直接写回 Laya 的材质参数。系统必须既能利用规则和 shader 先验，又要能自动探索大参数空间，并为后续机器学习和大模型辅助预留接口。

7.1 算法方案选择原则

效果优先

最终以真实 Laya 多视角渲染效果为准。算法选择不能只看收敛曲线或单项指标，必须能稳定产生视觉上接近的候选。

自动化优先

人工可以用于定义目标、校准偏好和验收 Top-K，但不应成为每轮优化必需环节。系统应自动完成参数开放、搜索、验证和回写。

可扩展优先

第一版不必引入所有高级算法，但架构上要能自然接入代理模型、历史经验库、学习式初始化和大模型 shader 分析。

7.2 可选算法路线调研

算法路线能解决什么优势风险/不足本项目定位
规则映射 / 规则优化Unity 参数到 Laya 参数的初始迁移、shader 语义解析、默认范围设定。稳定、快、可解释，适合形成先验。无法处理跨引擎光照/BRDF/采样差异，单独使用上限低。必须有，作为初始化和搜索空间生成。
网格 / 随机 / Sobol / Latin Hypercube全局粗探索、敏感度分析、寻找多个起点。简单可靠，可复现，不依赖模型假设。高维效率低，不能长期作为主优化器。适合做 warmup、probe 和参数重要性分析。
Optuna / TPE混合参数、离散开关、条件参数和 trial 管理。工程成熟，适合 define、枚举、连续/离散混合搜索。在强耦合连续高维空间里不一定最强。核心组件，负责实验调度和条件搜索。
CMA-ES连续强耦合参数的黑盒优化。不需要梯度，适合 shader 参数互相补偿的情况。评估次数较多，维度太高时成本上升。核心组件，用于连续参数组和联合微调。
Powell / Nelder-Mead局部精修、小维度局部搜索。实现简单，适合接近好解后的微调。全局能力弱，容易局部最优。辅助，作为 refine 阶段算法。
Differential Evolution / Evolution Strategy全局搜索、非凸空间探索。鲁棒，能跳出部分局部最优。渲染次数成本高。可选，用于小参数集或疑难材质。
Bayesian Optimization / GP昂贵函数的少量参数优化。样本效率高，理论清晰。高维、条件参数和噪声场景会变困难。局部可用，不建议作为全局主力。
Pareto / NSGA-II颜色、亮度、高光、纹理等目标冲突时输出候选集合。不会被单一总分误导，适合 Top-K 选择。结果解释和 UI 展示更复杂。中期核心，用于多目标候选管理。
代理模型 Surrogate学习参数到分数/指标的近似关系，减少真实渲染次数。长期可显著加速搜索。需要历史数据，跨 shader 泛化风险高。中长期增强，先做数据采集接口。
学习式初始化 Learned Initializer从 Unity 参数、参考图和 shader 语义预测初始 Laya 参数。能减少冷启动成本，越用越强。需要大量成功案例，早期容易误导。长期目标，不能替代真实渲染验证。
可微渲染 / 神经逆向渲染通过可微 proxy 估计材质参数。研究价值高，可能提供强初始化。Laya 真实渲染不可微，proxy 与真实引擎存在 domain gap。长期研究，只适合做辅助初始化。
大模型辅助理解 shader、生成参数分组、解释失败、生成搜索空间。擅长语义分析和工程知识整合。不能稳定直接输出最优浮点参数，存在幻觉风险。辅助智能，不作为数值优化主循环。

7.3 不采用单一算法的原因

不推荐：单算法押注

只用规则映射 只用 TPE 只用 CMA-ES 只用神经网络 只让大模型猜参数

问题：规则上限低；TPE/CMA-ES 各有盲区；神经网络早期没有数据；大模型不适合稳定数值寻优。

推荐：分层混合优化

规则先验 + 自动参数画像 全局粗探索 + 敏感度分析 TPE 管理条件参数 CMA-ES 优化连续耦合参数 分组交替 + 全参数联合微调 Top-K / Pareto / holdout 验证

各算法承担不同角色，最终统一由真实 Laya 渲染和多视角指标验收。

7.4 拟采用的自动化主方案

1输入解析读取 shader、lmat、Unity 参数、参考图、metadata

7.7 与行业研究的关系

逆向渲染和材质估计研究通常使用 analysis-by-synthesis：给定参数渲染图像，与目标图像比较，再优化参数。游戏工业中的材质迁移则更多依赖规则、预设库、人工经验和回归测试。本项目应采用二者的中间路线：短期以真实 Laya 渲染闭环和黑盒优化为核心，长期用历史 trial 数据训练代理模型和初始化模型，但任何学习模型的输出都必须经过 Laya 真渲染验证。

7.8 阶段性路线图

P1 稳定泛化覆盖更多材质类型和模型，减少失败率。分组交替、joint refine、自动扩边、Pareto、holdout views。不同视角和不同模型上都不依赖单视角作弊。

08. 强耦合 Shader 参数优化：不要把“分阶段”理解成单向锁死

关键修正：对于 Custom_low.shader 这种参数多、乘加混合、包含 ramp / specular / rim / emission / HSV 后处理的 shader，简单的「先调颜色 → 固定颜色 → 再调高光 → 固定高光」很容易卡在局部最优。分阶段只能作为初始化、诊断和降维手段，不能作为永久锁死流程。

8.1 为什么这个 shader 参数会互相影响

当前 shader 的最终颜色大致由这些部分叠加和变换而来：

texCol = pow(MainTex, u_TexPower) SSA = pow(LMap, u_GammaPower) diffuse = mix(ShadowColor, u_Color, ramp(NdotL)) * texCol specular = SpecularColor * specularRamp * SpeTex * SpecularIntensity ao = mix(1.0, LMap.b, u_AoPower) SkyColor = CubeReflection * u_IndirectStrength * u_ReflectColor * LMap.g result = (diffuse + specular) * ao * lightColor + SkyColor result += rim + emission result = HueShift / Contrast / Saturation 后处理

容易互相抵消的参数

  • u_Color、u_TexPower、u_Contrast 都可能改变整体亮度/颜色。
  • u_ShadowColor1、u_AoPower、u_LightRotate 都可能改变暗部表现。
  • u_SpecularIntensity、u_SpecularPower、u_SpecularThreshold 都可能改变高光强弱和面积。
  • u_HueShift、u_Saturation、u_Contrast 是全局后处理，容易掩盖前面材质参数错误。

错误的优化现象

  • 第一阶段颜色接近，但后续阴影/高光一调，颜色又偏。
  • 优化器用高光或自发光“作弊”弥补漫反射错误。
  • 某个视角很好，但侧面或俯视角明显变差。
  • 总分下降，但 highlight_loss 或 MS-SSIM 变坏。

8.2 推荐：分组交替优化 + 联合微调 + 回滚

不推荐：单向流水线

颜色组优化 → 永久固定 阴影组优化 → 永久固定 高光组优化 → 永久固定 最后全局调色

问题：后面参数会破坏前面结果，且前面错误可能被错误参数补偿。

推荐：Block-wise Alternating Optimization

for round in 1..R: 优化 base 组 优化 shadow/AO 组 优化 specular 组 优化 reflection 组 小范围开放 post-color 组 joint refine 联合微调 如果关键指标恶化则 rollback

每个参数组都可以被反复回访，最终以完整多视角 loss 决定是否接受。

09. 参数范围不是绝对真理：Legal Range、Soft Prior 与自适应扩边

shader 参数范围不应被写成一个固定死区间。更合理的是把范围拆成三层：legal_range、prior_range 和 trust_region。

Legal Range合法范围：shader/引擎允许的绝对边界，不能越界。
Prior Range先验范围：初始相信的合理区域，不是硬限制。
Trust Region信赖域：当前局部优化半径，可根据证据扩大或缩小。
History Centers历史中心：人类工程师确认过的有效值，作为新的搜索中心。

9.1 Soft Prior：允许极端值，但要有足够证据

total_loss = render_loss + λ * distance_from_prior(parameter)^2 + anti_cheating_penalty + view_stability_penalty

如果某个参数从 10 扩到 24 后，多视角渲染明显更接近 Unity，即使数值看起来偏大，也应该允许；但如果只是轻微改善，则正则项会让系统倾向保守解。

9.2 自适应扩边规则

触发条件含义动作
最优值贴近边界例如范围 [-10,10]，结果总在 9.8。执行边界外 probe。
边界外 probe 继续变好测试 12、15、20 后 loss 继续下降。扩展 trust region。
多个起点都推向同一边说明当前范围可能确实太小。扩大 prior 或增加新中心。
验证视角也变好不是只过拟合训练视角。接受扩边结果。

9.3 推荐参数配置结构

{ "name": "u_LightRotateY", "type": "float_periodic", "legal_range": [0, 360], "default": 0, "centers": [0, 24], "initial_radius": 10, "max_radius": 180, "expand_factor": 2.0, "regularization": { "mode": "distance_to_nearest_center", "weight": 0.05 }, "expand_when": { "near_boundary": true, "boundary_improves": true, "validation_required": true } }

9.4 没有人类经验的参数怎么办

  • 先用 shader 声明范围作为 legal_range。
  • 默认值或 Unity 映射值作为初始中心。
  • 做少量 probe：默认附近、小/中/大几个代表值、合法极值。
  • 如果 probe 显示它几乎不影响图像，先冻结。
  • 如果 probe 发现极端值明显改善，则把该值加入 centers，再进入正式优化。

10. 业界与研究领域常用方法

本项目可以被看作 Appearance Matching、Material Parameter Estimation、Analysis-by-Synthesis 和 Black-box Optimization 的组合问题。与论文中常见的「从照片恢复 PBR 材质」不同，我们不是追求物理参数的唯一真值，而是在 Unity 参考图和 Laya 真实渲染器之间寻找视觉等价的 shader 参数。因此，外部研究只能作为指标和优化策略参考，不能直接照搬。

10.1 指标研究的参考脉络

方向代表指标/思想别人怎么用我们如何采用
传统像素误差MSE, MAE, PSNR图像重建、压缩、超分辨率中常作为基础指标。保留 MAE 作为工程辅助，不把 PSNR/MSE 作为主分，因为它们对人眼色差和高光结构不敏感。
结构相似性SSIM, MS-SSIM衡量局部对比、结构和亮度变化，常用于图像质量评价。当前采用亮度通道 SSIM-L，并在 core_mask 内加权平均；MS-SSIM-L 可作为后续增强。
颜色科学CIE Lab, ΔE76, ΔE94, CIEDE2000显示、印刷、涂料和工业色差评估中常用。采用 ΔE00 作为颜色准确性主指标，并输出 mean 与 p95，避免平均色差掩盖局部严重偏色。
渲染图感知误差FLIP, HDR-VDP, error heatmap渲染器回归测试中用于定位肉眼可见差异。当前已实现 FLIP-like 轻量近似用于辅助展示；官方 FLIP 可作为后续 P2 增强。
深度感知指标LPIPS, DISTS生成图像、纹理重建、超分辨率中常用来近似人类主观判断。作为感知辅助分和报告展示，不作为唯一优化目标，避免模型偏好和环境依赖影响可复现性。
逆向渲染损失photometric, silhouette, perceptual, regularization通过多视角渲染误差反推材质、光照或几何参数。采用 multi-view + alpha mask + 参数正则化；但最终始终以 Laya 真渲染结果闭环。

10.2 优化方法的参考脉络

方法中文解释适合程度在本项目中的建议
Analysis-by-Synthesis分析-合成：猜参数、渲染、比较、再调整。很适合作为 Unity → Laya 外观匹配的主框架。
CMA-ES协方差矩阵自适应进化策略，适合强耦合连续参数。很适合用于 10~25 个关键连续参数的精优化。
Optuna / TPE工程化超参数搜索，适合连续/离散/条件参数。很适合管理 shader variant、trial、日志和搜索空间。
Block Coordinate Descent分块坐标下降：按参数组轮流优化，反复回访。很适合解决参数太多、全部一起搜困难的问题，但不能一组锁死后永不回访。
Bayesian Optimization贝叶斯优化，用代理模型决定下一次试哪里。中等适合适合少数昂贵参数，不适合直接 40 维全搜。
NSGA-II / Pareto多目标进化优化，输出一组互不完全支配的候选。适合用于颜色/高光/稳定性/参数保守性冲突时。
Differentiable Rendering可微渲染，通过梯度反推材质/光照/几何。长期适合可做近似 proxy 初始化，但最终仍需 Laya 真渲染校正。
Surrogate Model代理模型，用机器学习近似参数到 loss 的关系。中长期适合积累 trial 后训练，用于减少真实渲染次数。
Learned Initializer学习式初始化：神经网络从图像预测初始参数。长期适合需要大量历史数据，适合作为后续增强。
Human-in-the-loop人在环优化，人类从 Top-K 候选里选择，系统继续搜索。很适合用于主观风格判断和极端参数合理性确认。

10.3 我们和外部研究的关键区别

跨引擎，不是同引擎重建

Unity 和 Laya 的 BRDF、贴图采样、色彩空间、灯光模型和后处理可能不一致。我们的目标是最终图像外观接近，不是恢复 Unity 中每个物理参数的真实值。

可控场景，不是自然照片

我们有固定模型、固定相机、固定灯光和透明背景，因此可以强依赖 alpha mask、多视角命名和视角 metadata，避免自然图像评价中的很多不确定性。

调参需要解释，不只是排名

论文指标常只关心谁更像；本工具还要告诉用户为什么不像，以及应该优先调整哪一组 Laya shader 参数。

10.4 推荐技术路线

P0：真实渲染闭环 + 规则初始化 + 参数画像 + Sobol/LHS 粗探索 + TPE/CMA-ES 自动优化 P1：分组交替 + 全参数联合微调 + 自适应扩边 + Pareto/Top-K + holdout views 验证 P2：历史 trial 数据集 + Surrogate Model + Learned Initializer + 参数重要性预测 P3：大模型 shader 分析 + 自动搜索空间生成 + 失败归因 + 人类偏好校准
最务实组合：第一版就应保持自动化主循环，而不是退化成人工半自动调参。Optuna/TPE 管理离散 define、条件参数和 trial；CMA-ES 优化连续强耦合参数；Block-wise Alternating Optimization 与 Joint Refine 兼顾降维和全局一致性；Multi-view Validation 防止单视角作弊；历史数据逐步训练 Surrogate Model 和 Learned Initializer。评价上使用 ΔE00 + luminance/SSIM-L + 条件式 highlight + 多视角 mean/p90/max 聚合，避免被单个指标误导。

13. 第一版落地建议

第一版不要过度复杂。 推荐先实现 P0：多视角、alpha mask、有效性检查、ΔE00、luminance、SSIM-L、research_loss_v1 和分层 JSON/UI 展示。highlight、gradient、FLIP/LPIPS/DISTS 可作为 P1/P2 逐步加入。

推荐 Pipeline

1. Unity 输出 reference 多视角 RGBA + metadata 2. Laya 输出 candidate 多视角 RGBA + metadata 3. 读取 alpha，生成 core_mask / full_mask 4. 执行 mask_iou 和 bbox sanity check 5. 在 core_mask 内计算 ΔE00、luminance、SSIM-L 6. 按条件启用 highlight / gradient / perceptual 指标 7. 对每个视角输出 validity、scientific、conditional、diagnostic 8. 仅聚合有效视角，使用 mean / p90 / max 得到 final_score 9. 输出 JSON、CSV、HTML 报告和误差热力图 10. UI 展示总分、分层分数、每视角指标和通道诊断

最终建议

  • 预处理不做复杂配准，但保留 alpha mask、颜色空间、透明边缘和对齐检查。
  • 视角使用 12 张：水平 8 张 + 上斜 4 张；优化阶段可先用 8 张。
  • 主指标使用 ΔE00 + luminance + SSIM-L；highlight、gradient、LPIPS/FLIP/DISTS 作为条件或辅助指标。
  • 多视角聚合使用 mean + p90 + max，避免单个差视角被平均掩盖。
  • 后续自动优化建议采用黑盒优化；分阶段只用于初始化和诊断，不应单向锁死，应采用分组交替优化、联合微调和回滚验证。
